# O/A Level Resources — Project Overview

## Architecture
- **Monorepo** managed with pnpm workspaces
- **API server**: `artifacts/api-server/` — Express 5, TypeScript, Node.js 24, Neon PostgreSQL (raw `pg` pool, not Drizzle)
- **Admin dashboard**: `artifacts/admin/` — React 19, Vite, Tailwind v4, shadcn/ui, TanStack Query, Wouter
- **Android app**: `android/` — Kotlin, Jetpack Compose; NOT buildable in Replit, requires Android Studio
- **Shared libs**: `lib/api-spec/openapi.yaml` (source of truth), `lib/api-client-react/src/generated/` (do not edit — generated by Orval)

## Key conventions
- After editing `lib/api-spec/openapi.yaml`, run: `pnpm --filter @workspace/api-spec run codegen`
- DB is Neon PostgreSQL (cloud); `DATABASE_URL` env var; migrations run at startup in `artifacts/api-server/src/db/migrate.ts`
- Config (API keys, AI model, etc.) stored in a `config` DB table as key-value strings, not env vars
- JWT tokens include a `ver` counter — logout/password-change increments `jwtVersion` in DB to invalidate all prior tokens
- `ADMIN_DEFAULT_PASSWORD` env var required for first login only; once password is set via UI it's no longer read

## Deployment
- Render.com via `render.yaml` — NOT configured for Replit deployment
- Admin dashboard is served as static files from the API server in production (`artifacts/admin/dist/public/`)

## What works in Replit dev
- API server on port 3001, admin dashboard on port 5000
- Needs `DATABASE_URL` (Neon), `ADMIN_DEFAULT_PASSWORD` to be functional
